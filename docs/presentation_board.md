# Concept (コンセプト)
**「物流を、もっとスマートに。」**
日本の複雑な配送料金体系を解決する、次世代の配送計算エンジン「SmartShip」。
単純なサイズ計測ではなく、高度なアルゴリズムで「最適な箱詰め」をシミュレーションし、誰でも迷わずに一番安い発送方法を選べる体験を提供します。

# 制作背景 (Background)
*   **複雑すぎる料金体系**: 日本の宅配便（ヤマト、日本郵便など）は、サイズ（60/80/100）や重量、厚さ制限（ネコポス、宅急便コンパクトなど）が複雑に組み合わさっています。
*   **「早見表」の限界**: 既存の「配送方法早見表」は条件分岐が多く、初心者が最適なひとつ即座に選ぶのは困難です。
*   **空間把握の難しさ**: 単純な3辺合計（縦+横+高さ）の計算では、商品の形状や「隙間」を考慮できないため、「入ると思ったのに入らない」や「無駄に大きな箱を使ってしまう」という問題が多発しています。

# 解決策 (Solution)
*   **3Dパッキングアルゴリズム**: 業界標準のライブラリ（skjolber/3d-bin-container-packing）を拡張し、数千通りの配置パターン（回転を含む）を総当たりで計算。テトリスのように隙間なく詰めた場合の「真の最小サイズ」を導き出します。
*   **リアルタイム3D可視化**: 計算結果をブラウザ上で瞬時に3Dモデル化。参照オブジェクト（スマホや缶ジュース）と並べて表示することで、サイズ感を直感的に伝えます。
*   **不採用理由の透明化**: 「なぜこのプランが選ばれなかったのか（例：厚さが2mmオーバー）」を具体的に提示し、ユーザーの納得感と学習をサポートします。

# セールスポイント (Sales Point)
1.  **確実なコスト削減**: 人間の勘に頼らず、アルゴリズムが「入る最小の箱」を科学的に選定するため、配送料の無駄をゼロにします。
2.  **Visual Confidence (見てわかる安心感)**: 静止画ではなく、実際に動かせる3Dシミュレーションにより、実際に梱包する前の不安を解消します。
3.  **圧倒的なUX**: モダンなデザインとスムーズなインタラクションで、面倒な配送計算を「楽しい体験」へと変えました。
4.  **技術的な信頼性**: バックエンド（Java/Spring Boot）による堅牢な計算処理と、フロントエンド（React）による柔軟な表現力を組み合わせた、本格的な技術構成です。
